## Spacing Scale

This project uses SwiftUI system layout primitives. All explicit spacing must use multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon-to-label gaps within a row |
| sm | 8pt | Compact inline padding |
| md | 16pt | Default Form section inset (SwiftUI default) |
| lg | 24pt | Section header to first row gap |
| xl | 32pt | Not used in Phase 2 |
| 2xl | 48pt | Not used in Phase 2 |
| 3xl | 64pt | Not used in Phase 2 |

Exceptions:
- Touch targets: minimum 44pt height on all tappable rows (SwiftUI Form rows satisfy this automatically)
- "Add Contact" button row: minimum 44pt touch target height enforced by Form row layout

**Source:** iOS HIG minimum touch target 44×44pt; SwiftUI Form row defaults; 8-point grid applied to iOS pt units.

## Typography

SwiftUI semantic text styles — do not hard-code point sizes. These map to Dynamic Type and respect user accessibility settings.

| Role | SwiftUI Style | Weight | Usage |
|------|--------------|--------|-------|
| Body | `.body` | Regular (400) | TextField labels, linked contact row text, phone number field |
| Label | `.caption` | Regular (400) | Secondary metadata (gender matching label in PlayerRow — existing pattern) |
| Heading | `.headline` | Semibold (600) | Form section headers ("Player Info", "Contacts") — system default |
| Navigation Title | `.largeTitle` / `.inline` | Bold (700) | "Edit Player" / "Add Player" navigation title — existing pattern |

Line heights are controlled by SwiftUI's Dynamic Type rendering. Do not override `lineSpacing` on any text in this phase.

**Source:** Existing `PlayerFormView.swift` and `RosterView.swift` — `.font(.caption).foregroundStyle(.secondary)` pattern; SwiftUI system defaults.

## Color

This project uses SwiftUI semantic colors exclusively. Do not use hex values. Semantic colors adapt to light/dark mode automatically.

| Role | SwiftUI Color | Usage |
|------|--------------|-------|
| Dominant (60%) | `.systemGroupedBackground` | Form background (SwiftUI Form default) |
| Secondary (30%) | `.secondarySystemGroupedBackground` | Form section cells/rows (SwiftUI Form default) |
| Accent (10%) | `.accentColor` (system blue) | "Add Contact" button label and plus icon only |
| Destructive | `.red` (system red) | Swipe-to-delete action on linked contact rows only |
| Secondary text | `.secondary` foreground style | "Linked Contact" placeholder row text |

Accent reserved for: the "Add Contact" button row label and its `plus` SF Symbol — no other elements in this phase.

Destructive color reserved for: the swipe-to-delete action background on linked contact rows — no other elements in this phase.

**Source:** CONTEXT.md D-03 (plus icon button), D-06 (swipe-to-delete); existing `RosterView.swift` `.foregroundStyle(.secondary)` pattern; iOS HIG destructive action color.

## Screen and State Inventory

### PlayerFormView (modified)

| State | UI |
|-------|----|
| No linked contacts, form is clean | Section("Contacts") shows only the "Add Contact" button row |
| One or more linked contacts | ForEach rows above the "Add Contact" button row, one row per identifier |
| Contact picker open | Sheet presented over form — full-screen native CNContactPickerViewController |
| Contact picker cancelled | Sheet dismissed, contactIdentifiers unchanged |
| Contact selected | Sheet dismissed, new identifier appended (if not duplicate), row appears |
| Swipe-to-delete initiated | System swipe reveals red delete affordance (SwiftUI .onDelete default) |
| Delete confirmed | Row removed, identifier removed from contactIdentifiers |

**Source:** CONTEXT.md D-03 through D-06; RESEARCH.md Pattern 2.

## Interaction Contract

### "Add Contact" button row

- Placement: Last row in Section("Contacts"), always present regardless of linked contact count
- Label: `Label("Add Contact", systemImage: "plus")`
- Color: `.accentColor` (system blue) on label and icon
- Tap action: Sets `showContactPicker = true`, which triggers `.sheet(isPresented: $showContactPicker)`
- No disabled state — always tappable

### Contact picker sheet

- Presented as: `.sheet(isPresented: $showContactPicker)` containing `ContactPickerRepresentable`
- `ContactPickerRepresentable` returns `UINavigationController(rootViewController: picker)` — not bare `CNContactPickerViewController`
- On contact selected: identifier appended if not already in `contactIdentifiers`, sheet dismissed (`showContactPicker = false`)
- On cancelled: picker self-dismisses, no state change
- No authorization prompt before sheet — `CNContactPickerViewController` is out-of-process and requires none

### Phone number TextField

- Placement: Last row in Section("Player Info"), after Gender/DefaultMatching pickers
- Placeholder: "Phone"
- Keyboard type: `.phonePad`
- State management: `@State private var phoneNumber: String?` — initialized from `player?.phoneNumber` in `.onAppear`, written back in `save()`

### Linked contact rows

- Each row: `Text("Linked Contact").foregroundStyle(.secondary)`
- No navigation link — rows are display-only in Phase 2
- Swipe-to-delete enabled via `.onDelete` on the `ForEach`
- Removal updates local `contactIdentifiers` state; persisted only on "Save"

### Save behavior

- Same as existing: `save()` writes all local `@State` back to the model and calls `dismiss()`
- `contactIdentifiers` local state is written to `player.contactIdentifiers` in `save()`
